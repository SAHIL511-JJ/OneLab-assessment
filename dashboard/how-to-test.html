How to Test

Use this checklist while testing the app.

Step 1: Start

  • Open the dashboard page.
  • Choose one option: upload your files or click Use Sample Data.
  • If you do not have files, use the dashboard Download Test CSVs buttons.
  • If you upload files, upload both CSVs.
  • Transactions CSV = platform records, Bank Settlements CSV = bank records.

Step 2: Map columns

  • Check the suggested columns in the mapping panel.
  • Fix any wrong dropdown selection.
  • Click Apply Mapping.
  • Wait for Mapping applied and preflight success.
  • Set row mismatch tolerance in Advanced Settings (default: 0.02).

Step 3: Run reconciliation

  • Click Run Reconciliation.
  • See the summary cards at the top.
  • Use tabs to inspect each discrepancy type.
  • Read the bold Total mismatch amount note under summary cards (Mismatch rows only).
  • Tiny row drift up to your tolerance is not marked as mismatch and is tracked in totals.

What each tab means

  • Cross-Month: txn and settlement are in different months.
  • Amount Mismatch: row difference is above tolerance.
  • Duplicates: same transaction ID appears more than once.
  • Orphan Refunds: settlement has no matching transaction row.

How tolerance works

  • For each matched row, app calculates: abs(expected - actual).
  • If difference is less than or equal to tolerance, row is not marked as mismatch.
  • If difference is greater than tolerance, row appears in Amount Mismatch.
  • Those small tolerated differences are added in the Rounding summary totals section.
  • 0.00 = strict mode, 0.02 = balanced default, 0.05 = lenient mode.

Required fields

  • Transactions CSV: transaction ID, transaction date, expected/net amount.
  • Settlements CSV: transaction ID, settlement date, settlement amount.
  • Other fields are optional and auto-detected when possible.

If something fails

  • Check missing or wrong column mapping first.
  • Fix invalid date or amount values in your CSV.
  • If row mismatches are zero, still check Rounding summary for total drift.
  • For stricter checking, lower tolerance and run reconciliation again.
  • Upload both files again and rerun.
